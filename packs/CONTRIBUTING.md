# 投稿梗剧本

一套剧本 = 一个 YAML 文件。梗库是这个项目的核心资产，你的品味就是产品。

## 格式

```yaml
id: your-pack-id          # 小写字母/数字/连字符，唯一
name: 剧本名               # 显示名，2-6 个字最佳
description: 一句话描述    # 会出现在选择卡片和 pack 页面上
version: 1
language: zh

style: |
  # 全局风格块：每张 prompt 都会注入，是整套一致性的来源。
  # 必须包含：Q版贴纸风格基底、纯白背景、人物特征与参考照片一致（含服装）、
  # 文案渲染要求。可以加你这套独有的氛围（配色/元素/情绪）。

memes:
  - id: meme-id            # 套内唯一
    caption: 文案           # 渲染在图内的字，2-6 字最佳，太长会糊
    expression: 表情描述    # 具体、可画。"麻木的职业假笑" > "开心"
    action: 动作描述        # 戏剧化、有画面感
    shot: 半身              # 头部特写 / 头肩特写 / 半身 / 全身
    motion: 可选            # 动图的运动描述；不写则从 action 推导
```

## 规则

1. **16 个梗一套**，前 8 个是免费层——把发送频率最高、最"想立刻用"的放前面
2. **文案用真实聊天语境的高频词**，不要自造梗；想象它出现在你自己的群里
3. **expression/action 要具体可画**：模型照着画，写得越戏剧化效果越好
4. 顺序就是叙事：有起承转合的套装比散装强
5. 提交前跑 `uv run biaoqingbao validate packs/你的文件.yaml`

## 什么样的剧本会被合并

- 语境清晰：这套是发给谁的？什么场合？（参考现有 6 套的分工）
- 节令限定欢迎：春节、高考、年终述职……
- 梗要"已经在流行"，不要"希望它流行"
